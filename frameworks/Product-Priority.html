Priority Frameworks

RICE Framework

A quantitative prioritization framework that helps teams evaluate initiatives based on four factors.

RICE Score Formula

RICE Score = (Reach × Impact × Confidence) ÷ Effort

  • Higher RICE score = Higher priority
  • All components should use consistent time periods (e.g., per quarter)
  • Final score helps compare different initiatives objectively

R - Reach

Number of people/events impacted in a specific time period

Example: 500 users per month

I - Impact

Effect on target metric (Scale: 0.25-3)

  • 3.0 = Massive impact
  • 2.0 = High impact
  • 1.0 = Medium impact
  • 0.5 = Low impact
  • 0.25 = Minimal impact

C - Confidence

Confidence level in estimates

  • 100% = High confidence
  • 80% = Medium confidence
  • 50% = Low confidence

E - Effort

Estimated person-months required

Example: 2.5 person-months

Example Calculation

Feature: Add search functionality

  • Reach: 5000 users/quarter
  • Impact: 2.0 (High impact)
  • Confidence: 80% (0.8)
  • Effort: 3 person-months

RICE Score = (5000 × 2.0 × 0.8) ÷ 3 = 2666.67

MoSCoW Framework

A prioritization method used to quickly identify what's critical versus what's desirable.

Must Have

Critical requirements that must be delivered

Example: Core authentication system

Should Have

Important but not vital features

Example: Advanced search filters

Could Have

Desired but not necessary features

Example: Theme customization

Won't Have

Features that won't be implemented in this phase

Example: Voice chat support

Value vs Effort Framework

A simple yet effective framework that evaluates tasks based on their value contribution and implementation effort.

High Value / Low Effort

Quick wins that should be implemented immediately

Example: Bug fixes, UI improvements

High Value / High Effort

Strategic projects that need careful planning

Example: Platform migration

Low Value / Low Effort

Tasks to consider when resources are available

Example: Minor feature enhancements

Low Value / High Effort

Tasks to avoid or postpone

Example: Nice-to-have complex features

Eisenhower Matrix

A time-management framework that helps prioritize tasks based on urgency and importance.

Important & Urgent

Tasks requiring immediate attention

Example: Critical bug fixes

Important & Not Urgent

Tasks to schedule for later

Example: Strategic planning

Not Important & Urgent

Tasks to delegate

Example: Routine maintenance

Not Important & Not Urgent

Tasks to eliminate

Example: Unnecessary meetings

Kano Model

A framework for prioritizing features based on customer satisfaction and feature implementation.

Basic Needs (Must-Have)

Features that must be present to avoid dissatisfaction

Example: Login functionality

Performance Needs

Features that increase satisfaction with better implementation

Example: Page load speed

Delighters

Unexpected features that create high satisfaction

Example: AI-powered recommendations

Indifferent & Reverse

Features that don't affect or negatively impact satisfaction

Example: Unnecessary complexity

ICE Scoring Framework

A quick prioritization framework based on Impact, Confidence, and Ease.

Impact

Potential effect on key metrics (Scale: 1-10)

Example: Revenue increase potential

Confidence

Certainty in estimates (Scale: 1-10)

Example: Based on market research

Ease

Implementation difficulty (Scale: 1-10)

Example: Technical complexity level

WSJF (Weighted Shortest Job First)

A framework that prioritizes jobs based on their cost of delay and job size.

Business Value

Direct customer or business benefit

Example: Revenue impact

Time Criticality

Urgency and time sensitivity

Example: Market window opportunity

Risk Reduction

Opportunity to reduce future risks

Example: Technical debt reduction

Job Size

Effort required to complete

Example: Development time estimate

Opportunity Scoring

A framework that identifies opportunities based on importance and satisfaction gaps.

Importance

How important is this feature to users?

Example: User survey results

Satisfaction

How satisfied are users currently?

Example: Current feature ratings

Opportunity Score

Importance - Satisfaction = Opportunity

Example: High importance, low satisfaction = High opportunity

HEART Framework

Google's framework for measuring user experience success.

Happiness

User satisfaction and attitude

Example: User satisfaction surveys

Engagement

User involvement and interaction

Example: Session duration

Adoption

New user acquisition

Example: New user signups

Retention

User return rate

Example: Monthly active users

Task Success

User goal completion

Example: Conversion rates

Eisenhower-MoSCoW Hybrid

A combined framework that merges urgency-importance with MoSCoW prioritization.

Must Have (Important & Urgent)

Critical features that need immediate attention

Example: Security patches

Should Have (Important & Not Urgent)

Important features that can be planned

Example: Performance optimization

Could Have (Not Important & Urgent)

Features that can be delegated

Example: Minor UI updates

Won't Have (Not Important & Not Urgent)

Features to eliminate or postpone

Example: Nice-to-have features
