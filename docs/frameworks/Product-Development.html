Agile Development

An iterative approach to project management and software development that helps teams deliver value faster and with fewer headaches.

Core Principles

Individuals and Interactions

Over processes and tools

Working Software

Over comprehensive documentation

Customer Collaboration

Over contract negotiation

Responding to Change

Over following a plan

Scrum Framework

A lightweight framework that helps people, teams, and organizations generate value through adaptive solutions for complex problems.

Scrum Roles

Product Owner

  • Defines product vision
  • Manages backlog
  • Sets priorities

Scrum Master

  • Facilitates ceremonies
  • Removes impediments
  • Coaches team

Development Team

  • Self-organizing
  • Cross-functional
  • Delivers increments

Scrum Events

Sprint Planning

Set sprint goals and select items from backlog

Daily Scrum

15-minute daily sync to plan work

Sprint Review

Demo completed work to stakeholders

Sprint Retrospective

Reflect and plan improvements

Lean Development

A methodology that focuses on eliminating waste and delivering value to customers as efficiently as possible.

Seven Wastes

Partially Done Work

Incomplete features or documentation

Extra Features

Features that aren't needed

Relearning

Not sharing knowledge effectively

Handoffs

Knowledge loss between teams

Delays

Waiting for approvals or resources

Task Switching

Context switching between work

Defects

Bugs and quality issues

Lean Principles

Eliminate Waste

Remove anything that doesn't add value

Build Quality In

Prevent defects rather than fixing them

Create Knowledge

Document and share learning

Defer Commitment

Make decisions at the last responsible moment

Deliver Fast

Reduce cycle time and batch size

Framework Comparison

Framework Best For Focus Area Team Size
Agile Adaptive planning Iterative delivery Small to medium
Scrum Complex projects Process framework 5-9 people
Lean Efficiency focus Waste elimination Any size
