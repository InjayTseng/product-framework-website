Product Prioritization Frameworks

Make better decisions with proven prioritization frameworks

RICE Framework

A quantitative prioritization framework that helps teams evaluate initiatives based on four key factors: Reach, Impact, Confidence, and Effort.

Scoring Components

Reach

Number of people/events impacted in a given time period

Example

500 users per quarter

Measurement

Users, transactions, or events

Impact

How much this will affect each person

0.25x

Minimal

1x

Medium

3x

High

Confidence

How confident you are in your estimates

100%

High confidence

80%

Medium confidence

50%

Low confidence

Effort

Time required from all team members

Person-Months

Total time across team

RICE Score = (Reach × Impact × Confidence) ÷ Effort

Kano Model

A model for feature prioritization based on customer satisfaction and investment in implementation.

Feature Categories

Basic Features

Must-have functionality that users expect

  • Expected by users
  • Cause dissatisfaction if missing
  • Basic functionality or existing
  • Don't increase satisfaction when present

Performance Features

Features that directly correlate with satisfaction

  • Better implementation = more satisfaction
  • Linear relationship with user satisfaction
  • Competitive differentiators

Excitement Features

Unexpected features that delight users

  • Unexpected delighters
  • No dissatisfaction if missing
  • High satisfaction when present

Indifferent Features

Features that don't significantly impact satisfaction

  • Low impact on satisfaction
  • Users don't care much
  • Consider avoiding

Value vs Effort

A simple but effective way to prioritize features by comparing their potential value against the effort required to implement them.

Quadrants

Quick Wins

High Value, Low Effort

  • Immediate priority
  • Easy to implement
  • Fast ROI

Major Projects

High Value, High Effort

  • Strategic initiatives
  • Careful planning needed
  • Long-term investment

Fill-ins

Low Value, Low Effort

  • Nice to have
  • Do when resources available
  • Quick improvements

Time Sinks

Low Value, High Effort

  • Avoid or postpone
  • Question necessity
  • Look for alternatives

Framework Comparison

Framework Best For Complexity Time Investment
RICE Data-driven decisions High Medium
Kano Model Feature classification Medium High
Value vs Effort Quick decisions Low Low
