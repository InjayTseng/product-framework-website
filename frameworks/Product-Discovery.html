Jobs To Be Done (JTBD)

A framework that focuses on understanding why customers "hire" your product to accomplish specific tasks or achieve certain outcomes.

Core Concept

People don't buy products; they hire them to get jobs done. Understanding these jobs helps create better solutions.

Key Components

  • Functional Job: The practical task the customer needs to accomplish
  • Emotional Job: How the customer wants to feel
  • Social Job: How the customer wants to be perceived
  • Job Statement: When [situation], I want to [motivation], so I can [expected outcome]

When to Use

  • Developing new products or features
  • Understanding customer motivation
  • Identifying unmet needs
  • Improving existing solutions

User Story Mapping

A visual exercise that helps teams understand the user journey and break it down into manageable pieces.

Map Structure

  • Backbone: High-level user activities
  • Walking Skeleton: Must-have user tasks
  • User Tasks: Detailed steps users take
  • Slices: Grouped releases of functionality
